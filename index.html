Election Algorithms

Distributed systems often require a central "coordinator" to manage tasks. When that coordinator fails, the remaining nodes must elect a new leader. Explore the two classic algorithms used to solve this problem dynamically.

Theory

The Ring Algorithm is used in systems organized as a logical ring. We assume unidirectional links, meaning each node only sends messages to its right neighbor.

When a process detects a coordinator failure, it creates an Active List starting with its own ID and sends an ELECTION message to the right. As the message traverses the ring, every active node appends its ID. Once the message completes a full circle, the initiating node examines the list and selects the highest ID as the new coordinator.

Example Execution

  1. Node 5 crashes (the current coordinator).
  2. Node 2 detects the crash and sends ELECTION[2] to Node 3.
  3. Node 3 appends its ID and forwards ELECTION[2,3].
  4. Node 4 appends its ID and forwards ELECTION[2,3,4].
  5. Node 1 appends its ID and forwards ELECTION[2,3,4,1].
  6. Node 2 receives its own message, picks the highest ID (4), and broadcasts COORDINATOR(4).

    Theory

    The Bully Algorithm relies on the assumption that every process can communicate with every other process. Nodes "bully" their way to the top.

    When a process detects failure, it sends an ELECTION message to all processes with higher IDs. If no one responds, it assumes the throne and broadcasts a COORDINATOR message. If a higher node responds, the lower node gives up, and the higher node takes over the election process until the absolute highest active node wins.

    Example Execution

    1. Node 5 crashes.
    2. Node 2 detects it, sends ELECTION to 3, 4, 5.
    3. Nodes 3 & 4 respond OK to 2. Node 2 stops.
    4. Node 3 sends ELECTION to 4, 5. Node 4 sends ELECTION to 5.
    5. Node 4 responds OK to 3. Node 3 stops.
    6. Node 4 receives no response from 5.
    7. Node 4 broadcasts COORDINATOR(4).
